# 💒 Wedding Invitation App

A modern, elegant, and fully dynamic wedding invitation web application built with **Next.js**, **Supabase**, and **Tailwind CSS**. Features a stunning black & white theme with gold accents, personalized guest links, RSVP system, and an admin dashboard.

## 🚀 Getting Started

### Prerequisites

- Node.js 18+
- Supabase account ([supabase.com](https://supabase.com))

### 1. Clone & Install

```bash
git clone https://github.com/carlosmarpaung08/wedding-invitation-app.git
cd wedding-invitation-app
npm install
```

### 2. Setup Supabase

Create a new Supabase project, then run this SQL in the **SQL Editor**:

```sql
-- Guests table
CREATE TABLE guests (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,
  slug TEXT UNIQUE NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- RSVP table
CREATE TABLE rsvp (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  guest_id UUID REFERENCES guests(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  is_attending BOOLEAN DEFAULT FALSE,
  total_guests INTEGER DEFAULT 1,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Wishes table
CREATE TABLE wishes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL,
  message TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable Realtime for wishes
ALTER PUBLICATION supabase_realtime ADD TABLE wishes;

-- RLS Policies
CREATE POLICY "Enable read access for everyone" ON "public"."guests"
  AS PERMISSIVE FOR SELECT TO public USING (true);

CREATE POLICY "Enable insert for everyone" ON "public"."guests"
  AS PERMISSIVE FOR INSERT TO public WITH CHECK (true);

CREATE POLICY "Enable delete for everyone" ON "public"."guests"
  AS PERMISSIVE FOR DELETE TO public USING (true);

CREATE POLICY "Enable insert for everyone" ON "public"."rsvp"
  AS PERMISSIVE FOR INSERT TO public WITH CHECK (true);

CREATE POLICY "Enable read access for everyone" ON "public"."rsvp"
  AS PERMISSIVE FOR SELECT TO public USING (true);

CREATE POLICY "Enable insert for everyone" ON "public"."wishes"
  AS PERMISSIVE FOR INSERT TO public WITH CHECK (true);

CREATE POLICY "Enable read access for everyone" ON "public"."wishes"
  AS PERMISSIVE FOR SELECT TO public USING (true);
```

### 3. Environment Variables

Create `.env.local`:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY=your_anon_key
```

### 4. Run Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) to see the app.

## 🎯 Usage

### Personalized Guest Links

Add guests via Admin Dashboard (`/admin`), then share their unique link:

```
https://your-domain.com/?to=guest-slug
```

### Admin Dashboard

Access at `/admin` with password: `admin123`

> ⚠️ Change the admin password in `src/app/admin/page.tsx` before deploying.

## 📦 Deployment

Deploy to [Vercel](https://vercel.com):

1. Push your code to GitHub
2. Import the repository on Vercel
3. Add environment variables (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY`)
4. Deploy!
